# Smart Contracts

## Why Contracts

Settlement between two parties who do not trust each other requires either a regulated
intermediary (bank, payment gateway) or a smart contract that enforces the terms programmatically.
For cross-border remittance, the intermediary route adds days and fees. A contract cuts both.

Four contracts, each scoped to a single concern.

## MultiCurrencyWallet.sol (ERC-1155)

**Why.** Tracks token balances per user address — how much USDC, how much synthetic INR — without
deploying individual ERC-20 contracts. ERC-1155 is a single contract that holds multiple token
types, which means one address lookup per user instead of one per currency.

**How.** `mint(to, tokenId, amount)` is called by the relayer when a user on-ramps. Token ID 1 is
USDC, token ID 2 is synthetic INR. The frontend reads `balanceOf(userAddress, tokenId)` via
ethers.js. No transfer logic lives here; transfer happens through the escrow contract.

Uses OpenZeppelin's ERC-1155 — no custom audit surface beyond the mint/burn role checks.

## RemittanceEscrow.sol

**Why.** An escrow contract solves the fundamental trust problem: the sender deposits funds and
sets a timelock. The receiver claims when conditions are met. If the receiver never claims, the
sender gets their funds back after the timelock expires. No intermediary needed.

**How.**

```
createRemittance(receiver, token, amount, lockPeriod)
  → pulls tokens from sender into contract
  → stores Escrow{ sender, receiver, token, amount, lockUntil, status }

release(escrowId)
  → called by receiver (relayer can call on their behalf)
  → pushes tokens to receiver
  → emits EscrowReleased

dispute(escrowId)
  → freezes escrow, pings off-chain AI resolver

refund(escrowId)
  → sender only, after lockUntil
  → returns tokens to sender
```

The timelock is Bitcoin's `OP_CHECKLOCKTIMEVERIFY` adapted to EVM: `lockUntil =
block.timestamp + lockPeriod`. The default period is 72 hours for MVP.

**Bitcoin pattern used:** CLTV-style absolute timelock on the refund path. The sender cannot
reclaim early; the receiver cannot hold funds hostage past the deadline.

## OracleProxy.sol

**Why.** Users need to know the INR value of their USDC balance and the app needs to lock an
exchange rate when a remittance is created. Chainlink's USD/INR feed is the canonical source on
Polygon.

**How.** Wraps `AggregatorV3Interface` at the Chainlink USD/INR oracle address for Polygon.
`getConversionRate()` returns `(rate, decimals)`. The backend fetches this when creating a
remittance and stores the rate alongside the escrow record so both sides know what rate was
locked.

Uses Chainlink's existing feed — no custom oracle logic, no upkeep costs beyond the gas for the
read.

## ZKVerifier.sol

**Why.** The app needs to verify age (≥18) and country of residence without storing the
underlying documents. A Groth16 verifier contract checks a zero-knowledge proof on-chain. If the
proof validates, the user's age or country was verified without the contract ever knowing their
DOB or ID number.

**How.** Generated from Noir circuits (see ZK.md). Implements `IVerifier` interface:
`verify(proof, pubSignals) → bool`. The contract is auto-generated by `nargo compile` — no manual
Solidity to audit.

## Deployment Order

1. MultiCurrencyWallet.sol (no dependencies)
2. OracleProxy.sol (needs Chainlink feed address)
3. ZKVerifier.sol (needs compiled verification key)
4. RemittanceEscrow.sol (needs MultiCurrencyWallet address)

All deploy via `forge script`. The `Deploy.s.sol` script reads addresses from environment
variables and writes deployed addresses to a JSON file that the backend reads at startup.

## Gas Estimates

| Operation | Gas (estimated) |
|-----------|----------------|
| createRemittance | ~80k |
| release | ~45k |
| refund | ~35k |
| dispute | ~25k |
| rate lookup | ~20k |

On Polygon ($0.01/tx average), remittance costs ~$0.0008. Even at mainnet peak ($0.05/tx), it is
~$0.004. Less than 0.1% of any remittance over $10.

## Cost to Deploy

About $2-3 in MATIC gas to deploy all four contracts. The professional audit is the real cost
($5k-15k), but that applies to the escrow and wallet contracts, not the oracle wrapper or
verifier (both are standard patterns).
